# Self-Hosted RDP Gateway

Веб-шлюз удалённого доступа на базе **Apache Guacamole**, развёрнутый в Docker Compose.
Подключение к удалённым машинам по **RDP / SSH / VNC прямо из браузера** — без VPN, без
установки клиентов, по защищённому HTTPS на собственном домене.

> Оригинал использует Nginx с самоподписанным сертификатом и доступен только по IP.
> В этой версии связка **переработана под реальный production-сценарий**: публичный домен,
> динамический DNS и автоматический валидный TLS.

## Что здесь сделано

По сравнению с исходным проектом:

- 🔁 **Nginx заменён на [Caddy](https://caddyserver.com/)** — реверс-прокси с автоматическим
  выпуском и продлением TLS-сертификатов Let's Encrypt. Самоподписанные сертификаты и ручная
  генерация ключей больше не нужны.
- 🌐 **Добавлен DuckDNS** — динамический DNS: контейнер держит A-запись домена в актуальном
  состоянии даже при «плавающем» белом IP (актуально для домашнего сервера/динамического IP
  провайдера).
- 🔒 **Корректная передача реального IP клиента** в Guacamole (`REMOTE_IP_VALVE_ENABLED`) —
  для правильного логирования и аудита подключений за прокси.
- 🗝️ **Все секреты вынесены в `.env`** (токен DuckDNS, пароль БД, домен). В репозитории
  лежит только `.env.example`.
- 🧹 Скрипты `prepare.sh` / `reset.sh` очищены от логики Nginx.

## Архитектура

```
                    Интернет
                       │  HTTPS :443 / HTTP :80
                       ▼
            ┌─────────────────────┐        ┌──────────────┐
            │        Caddy        │        │   DuckDNS    │
            │  reverse proxy +    │        │  обновляет   │
            │  авто-TLS (LE)      │        │  A-запись    │
            └──────────┬──────────┘        └──────────────┘
                       │ proxy :8080
                       ▼
            ┌─────────────────────┐
            │     Guacamole       │  веб-приложение (HTML5)
            └──────────┬──────────┘
              ┌────────┴────────┐
              ▼                 ▼
       ┌────────────┐    ┌────────────┐
       │   guacd    │    │ PostgreSQL │
       │ RDP/SSH/VNC│    │ настройки, │
       │   движок   │    │ пользователи│
       └────────────┘    └────────────┘

        Сеть Docker: guacnetwork_compose (bridge)
        Наружу открыты только порты Caddy (80/443).
```

## Стек

| Компонент      | Роль                                                        |
|----------------|-------------------------------------------------------------|
| **Caddy**      | Реверс-прокси, автоматический HTTPS (Let's Encrypt)         |
| **DuckDNS**    | Динамический DNS для домена `*.duckdns.org`                 |
| **Guacamole**  | Веб-интерфейс удалённого доступа (HTML5)                    |
| **guacd**      | Демон, реализующий протоколы RDP / SSH / VNC / Telnet       |
| **PostgreSQL** | Хранилище пользователей, подключений и записей сессий       |
| **Docker Compose** | Оркестрация всех сервисов                               |

## Предварительные требования

- Сервер с **Docker** и **Docker Compose**
- Открытые порты **80** и **443** (для валидации и работы TLS)
- Поддомен и токен на [DuckDNS](https://www.duckdns.org/) (бесплатно)

## Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone https://github.com/armaggedec/self-hosted-rdp-gateway.git
cd self-hosted-rdp-gateway

# 2. Настроить переменные окружения
cp .env.example .env
nano .env          # указать домен, поддомен/токен DuckDNS и пароль БД

# 3. Подготовить БД и папки (генерирует init/initdb.sql)
./prepare.sh

# 4. Запустить
docker compose up -d
```

После старта Caddy сам получит TLS-сертификат, и интерфейс будет доступен по адресу
`https://<ваш-домен>` (корень автоматически перенаправляется на `/guacamole/`).

**Учётные данные по умолчанию:** `guacadmin` / `guacadmin` — **смените сразу после входа.**

## Конфигурация (`.env`)

| Переменная           | Назначение                                              |
|----------------------|---------------------------------------------------------|
| `DUCKDNS_SUBDOMAIN`  | Поддомен DuckDNS (без `.duckdns.org`)                   |
| `DUCKDNS_TOKEN`      | Токен из личного кабинета DuckDNS                       |
| `DOMAIN`             | Полное доменное имя для TLS-сертификата                 |
| `POSTGRES_DB`        | Имя базы данных                                         |
| `POSTGRES_USER`      | Пользователь БД                                         |
| `POSTGRES_PASSWORD`  | Пароль БД                                               |
| `TZ`                 | Часовой пояс контейнеров                                |

## Как это работает

- **Caddy** слушает 80/443. Файл `Caddyfile` использует плейсхолдер `{$DOMAIN}`, который
  подставляется из переменной окружения. Caddy сам получает и продлевает сертификат
  Let's Encrypt; данные TLS хранятся в `./caddy_data` (поэтому их не стоит удалять при
  сбросе — иначе можно упереться в rate limit Let's Encrypt).
- **DuckDNS** периодически отправляет текущий IP сервера в API DuckDNS, поддерживая
  актуальность A-записи домена.
- **Guacamole** доступен наружу только через Caddy: порт `8080` опубликован лишь внутри
  сети Docker.
- **PostgreSQL** инициализируется скриптом `init/initdb.sql`, который генерирует `prepare.sh`.
  Данные сохраняются в `./data` между перезапусками.

## Скрипты

- **`prepare.sh`** — генерирует `init/initdb.sql` (схема БД Guacamole) и создаёт папку для
  записей сессий. Запускается один раз перед первым стартом.
- **`reset.sh`** — полный сброс: удаляет БД, записи и файлы дисков. TLS-данные Caddy
  сохраняются намеренно.

## Безопасность

- Секреты (`.env`) не хранятся в репозитории и добавлены в `.gitignore`.
- Весь внешний трафик идёт по HTTPS с валидным сертификатом.
- Наружу открыты только порты прокси (80/443); БД и guacd доступны только внутри сети Docker.
- Пароль `guacadmin` по умолчанию **обязательно** меняется после первого входа.
